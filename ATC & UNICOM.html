Air-to-Air (Uncontrolled Aerodromes) – ICAO UNICOM / AUTO-INFO ›
UNICOM (mostly FAA / USA term)
UNICOM = a ground station (often an FBO/handler) that can provide non-ATC information
	(local weather, runway in use, parking, fuel, etc.).

	Important: UNICOM is not a control service. At a non-controlled aerodrome, pilots self-announce
	on a traffic frequency (CTAF). Sometimes the CTAF is the same frequency as UNICOM depending on the field.

	The FAA also describes cases where UNICOM may relay certain messages for ATC if procedures exist
	(e.g., clearance via an operator), but this is controlled/limited by published arrangements.
ICAO / Europe / France (Auto-Info / Air-to-Air)
In France/Europe, the concept is usually “auto-information / air-to-air (A/A)”:
	at an aerodrome without ATS (no Tower, no AFIS), pilots broadcast position/intentions for
	collision avoidance — not to “receive an authorization”.

	Practical rule:
	• Use the published aerodrome frequency (VAC/AIP).
	• If none is assigned: use 123.500 MHz (common auto-information frequency).
	• For some ULM platforms, a specific framework exists: 125.335 MHz is referenced in AIC France (case dependent).

	Therefore:
	• USA: “UNICOM / CTAF”
	• France/ICAO: “Auto-info / A/A” on the aerodrome frequency (or 123.500 if none).
Operational goals
• Build a shared mental picture (“who is where, doing what, in which direction, on which runway”).
	• Avoid conflicts (joining, base/final, backtrack, runway occupancy, etc.).
	• Coordinate when there is no ATC (or the tower is closed).
	• Manage radio workload: short, clear, standard, and leave space on the frequency.
Message structure
A good transmission almost always contains:
	1) Aerodrome name (at the beginning AND often at the end if frequency is shared)
	2) Callsign
	3) Position + altitude (especially inbound)
	4) Intention
	5) Runway / circuit direction (left/right) / joining method

	Example (France/ICAO style):
	“Lognes traffic, N884CT, at parking, taxiing to holding point runway 26, for circuit, Lognes.”

	Example (USA/FAA style):
	“Frederick traffic, N884CT, 10 miles southeast, descending through 3,500, inbound for landing,
	request wind and runway, Frederick.”
Frequency discipline
• Listen before transmitting (2–3 seconds minimum).
	• If someone is talking: wait (avoid stepping on transmissions).
	• No chat: traffic only.
	• Always say the aerodrome name when the frequency can be shared (e.g., 123.500).
What auto-info is NOT
• You do not ask “cleared/authorized?”
	• You do not receive “cleared to land” (except very specific published services like AFIS/INFO —
	  but then it is no longer “pure UNICOM”).
Departure (ground)
• Before engine start / at parking (optional; useful with traffic/schools)
	• Taxi (especially if complex taxiways / runway crossings)
	• Holding point / lining up (critical)
	• Take-off (critical)
	• Leaving the circuit / frequency change (when departing the area)
Circuit
• Downwind
	• Base
	• Final
	• Runway vacated (important)
Arrival
• First inbound call (≈10 min / 10 NM / before joining)
	• Joining (overhead / downwind / straight-in / etc.)
	• Downwind / base / final
	• Runway vacated
	• Taxi to parking (optional but helpful)
Typical Frequencies (quick references) ›
France (practical references)
• Always prioritize the published VAC/AIP frequency.
	• If no frequency assigned: 123.500 MHz (common auto-information).
	• Special case: some ULM platforms use 125.335 MHz within the published framework.
USA (practical references)
• CTAF/UNICOM is commonly 122.7 / 122.8 / 122.9 / 123.0 (field dependent, published).
	• 122.8 is very common at non-towered fields.

	⚠️ You do not “choose” the frequency — you read it (chart / AFD / Chart Supplement / VAC).

Interactions & Traffic Management (ICAO – English) ›
Two aircraft converging on final
You do not negotiate, you declare:

	“Saint-Pierre traffic, N884CT, long final runway 26, I have the traffic in sight, continuing approach, Saint-Pierre.”

	If spacing is insufficient:

	“Saint-Pierre traffic, N884CT, going around for spacing, runway 26, Saint-Pierre.”
Someone backtracking the runway
“Saint-Pierre traffic, N884CT, holding short runway 26 due to backtrack traffic, Saint-Pierre.”

	Clear, calm, professional.
Adjust your plan based on traffic
This is very ICAO-realistic:

	“Saint-Pierre traffic, N884CT, extending downwind for spacing behind the reported traffic, Saint-Pierre.”

	or

	“Saint-Pierre traffic, N884CT, number two following the aircraft on base leg, Saint-Pierre.”
Priority messages (mental hierarchy)
• Emergency / abnormal
	• Runway occupancy (lining up, take-off, final, vacated)
	• Joining / circuit legs
	• Taxi / parking

	If in doubt: shut up and listen.

Quick Checklist (before every transmission) ›
Mental checklist
Before each call, check:
	• Aerodrome name (start + end if frequency is shared)
	• Callsign
	• Where am I (position/altitude)
	• What am I doing (intention)
	• Runway / circuit direction (left/right) / join type
	• Keep it short (one breath)
0. ATIS / Information Always first ›
The ATIS provides the current airport status and must be fully understood before contacting ATC. Your goal is to extract the information quickly, not to memorize the message.

Mental flow (always the same)

1. ATIS letter + time
	2. Runway(s) / airport configuration
	3. Wind
	4. Visibility + weather
	5. Clouds
	6. Temperature / Dewpoint
	7. Pressure (QNH / Altimeter)
	8. Special information / restrictions

What to listen for

• Information letter (Alpha, Bravo, India, November…)
	• Time (Zulu / UTC)
	• Runway(s) in use + approach type (ILS, RNAV, VOR…)
	• Wind (direction / speed / gusts / variable)
	• Visibility (and RVR if applicable)
	• Weather (rain, fog, snow, thunderstorms…)
	• Clouds (FEW / SCT / BKN / OVC + altitude)
	• Temperature and dewpoint
	• Pressure:
	  - Europe: QNH 1013
	  - USA: Altimeter 29.98
	• NOTAMs: taxiway closed, bird activity, wind shear, etc.
	• “Advise ATC you have Information ___”

Example ATIS broadcast

"All aircraft be advised, Information Juliet now current.
	Wind 230 at 8.
	Visibility 10.
	Temperature 20, dewpoint 11.
	Altimeter 29.98."

How a pilot decodes it

ATIS: Juliet
	Wind: 230° / 8 kt
	Visibility: 10
	Temperature: +20°C
	Dewpoint: +11°C
	Altimeter: 29.98
